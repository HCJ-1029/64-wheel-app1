## 📋 打包步骤（需要Java环境）

### 第一步：安装Java JDK

1. **下载Java JDK 17**（推荐Eclipse Temurin）
   - 访问：https://adoptium.net/temurin/releases/
   - 选择：Windows x64, JDK 17
   - 下载：`.msi` 安装包

2. **安装JDK**
   - 双击下载的 `.msi` 文件
   - 按照安装向导完成安装
   - **重要**：勾选 "Set JAVA_HOME variable"

3. **验证安装**
   打开新的命令提示符（CMD），输入：
   ```bash
   java -version
   javac -version
   ```
   应该显示版本信息（如 `17.0.x`）

### 第二步：安装Android Studio（如未安装）

1. **下载Android Studio**
   - 访问：https://developer.android.com/studio
   - 下载并安装

2. **安装Android SDK**
   - 打开Android Studio
   - 点击 "More Actions" → "SDK Manager"
   - 安装以下组件：
     - Android SDK Platform 34
     - Android SDK Build-Tools 34
     - Android SDK Platform-Tools

3. **设置ANDROID_HOME环境变量**
   - 默认路径：`C:\Users\你的用户名\AppData\Local\Android\Sdk`
   - 添加到系统环境变量 `ANDROID_HOME` 或 `ANDROID_SDK_ROOT`

### 第三步：构建APK

打开 **Git Bash**，执行：

## 🐛 常见问题

### 1. ERROR: JAVA_HOME is not set
**解决方案**：安装JDK并设置JAVA_HOME环境变量
- JAVA_HOME值示例：`C:\Program Files\Eclipse Adoptium\jdk-17.x.x.x-hotspot`

### 2. SDK location not found
**解决方案**：创建 `local.properties` 文件
```bash
# 在 64-wheel-app/android/ 目录下创建 local.properties
echo "sdk.dir=C:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk" > local.properties
```

### 3. 构建超时或失败
**解决方案**：
- 检查网络连接（需要下载Gradle依赖）
- 尝试清理后重新构建：
  ```bash
  cd android
  ./gradlew clean
  ./gradlew assembleDebug
  ```

## 📱 安装到手机

1. **启用开发者选项**
   - 设置 → 关于手机 → 连续点击"版本号"7次
   - 返回 → 系统和更新 → 开发者选项 → 开启"USB调试"

2. **安装APK**
   - 将 `app-debug.apk` 传到手机
   - 在手机上点击安装（需要允许"安装未知应用"）
